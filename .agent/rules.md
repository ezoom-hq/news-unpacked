# Project Rules

このファイル (`.agent/rules.md`) にプロジェクト固有のルールを記述すると、AI（Antigravity）はそれを読み込んで従うようになります。

### コードスタイルの指定
- コメントは日本語で記述すること

### 振る舞いの指定
- 回答は常に日本語で行うこと
- 修正案を提示する際は、必ずメリットとデメリットを併記すること
- セキュリティに関する警告は目立つように表示すること

### 禁止事項
- `any` 型の使用は極力避けること

### UI/UX デザイン原則 (UX心理学)
参考: [UX心理学 | グロース](https://www.shokasonjuku.com/ux-psychology)

1.  **美的ユーザビリティ効果 (Aesthetic-Usability Effect)**
    - 美しいデザインは使いやすいと感じられる。UIの見た目（配色、余白、フォント）には妥協せず、「プレミアム感」を重視すること。
2.  **ドハティの閾値 (0.4秒の壁)**
    - ユーザーのアクションに対するフィードバックは0.4秒以内に返すこと。
    - サーバー通信など時間がかかる処理でも、まずは楽観的UI（Optimistic UI）で即座に反応を返すこと（ボタンの凹み、ローディング表示など）。
3.  **ピーク・エンドの法則 (Peak-End Rule)**
    - 体験の「最高潮（議論の盛り上がり）」と「最後（サマリー表示）」が最も記憶に残る。
    - ゲーム終了画面や結果発表の演出は特にリッチにすること。
4.  **認知負荷 (Cognitive Load)**
    - 画面上の情報は必要最小限に抑え、ユーザーを迷わせないこと。
    - 複雑な手順（準備→議論）はウィザード形式などで段階的に見せること（段階的開示）。
5.  **フィードバックの徹底**
    - 全てのインタラクティブ要素（ボタン、入力欄）は、ホバー時・クリック時に明確な視覚的フィードバック（色変化、拡大縮小、波紋効果など）を持つこと。
